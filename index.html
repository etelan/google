Google Fit

android + wearables | front end

The first team I was placed upon was Google Fit on Android.
I initially worked on fixing Accessibility Bugs, for instance, talkback bugs.
Next, I worked on adding a Landscape Mode for Google Fit's Activity Tracking.

For this feature, I interacted with UI and QA in order to get my layout approved.
I then moved onto Google Fit On Wear, the WearOS Google Fit App.

For this application, a user facing bug was sent to me.
The daily goals were not being reset at midnight, like how users expect.

First of all, I manually tested the bug, by staying up to midnight and watching it not reset.
Then I wrote up a method for resetting the goals, which sets our local goal data store to zero progress.

I then worked on the unit test Finally I implemented this method to be called when the day changes.
This allows the goals to reset regardless of stuff like timezone changes.




Android Health Connect

I started on working on support for localdate times, for deleting and reading.
This allowed me to learn SQL, specifically for connecting the application to the SQLite database.

Next I worked on optimizing the sql queries we make.

For example, instead of reading all the data and then deleting them all individually,
I simply delete as I find the UUIDs to delete.

This allows us to do less reads on the database which speeds up execution time.
